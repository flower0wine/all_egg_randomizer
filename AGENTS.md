## Code Style Guidelines

### Formatting
- **Indentation**: 4 spaces (no tabs)
- **Line endings**: Native (LF on Linux/macOS, CRLF on Windows - Gradle handles this)
- **Line length**: Soft limit 120 characters
- **Braces**: K&R style (opening brace on same line)
- **Imports**: Sorted alphabetically, grouped by package type

### Naming Conventions
| Element | Convention | Example |
|---------|------------|---------|
| Package | lowercase, no underscores | `com.alleggrandomizer.mixin` |
| Class/Interface | PascalCase | `AllEggRandomizer`, `ExampleMixin` |
| Method | camelCase | `onInitialize()`, `loadLevel()` |
| Field/Variable | camelCase | `MOD_ID`, `logger`, `callbackInfo` |
| Constant | UPPER_SNAKE_CASE | `MOD_ID = "alleggrandomizer"` |
| Mixin Class | PascalCase, ends with Mixin | `ExampleMixin`, `PlayerMixin` |

### Java Best Practices
- Use diamond operator `<>` where type inference applies
- Prefer `var` for local variables when type is obvious
- Use `@Override` for all overridden methods
- Use `final` for method parameters and local variables where appropriate
- Avoid wildcard imports (`import java.util.*`)

### Error Handling
- Use SLF4J logger: `AllEggRandomizer.LOGGER`
- Log levels: `ERROR` > `WARN` > `INFO` > `DEBUG` > `TRACE`
- Wrap potentially risky code in try-catch when necessary
- Never swallow exceptions silently

## Mixin Development

### Mixin Naming
- Server mixins: `com.alleggrandomizer.mixin.*`
- Client mixins: `com.alleggrandomizer.mixin.client.*`

### Mixin Config Files
- Server: `alleggrandomizer.mixins.json` - includes `"compatibilityLevel": "JAVA_21"`
- Client: `alleggrandomizer.client.mixins.json`

### Mixin Patterns
```java
@Mixin(TargetClass.class)
public class MyMixin {
    @Inject(at = @At("HEAD"), method = "targetMethod")
    private void onTargetMethod(CallbackInfo ci) {
        // Code here
    }
}
```

### Common Injection Points
- `@At("HEAD")` - method start
- `@At("TAIL")` - method end
- `@At("RETURN")` - before return
- `@At("INVOKE")` - at method call
