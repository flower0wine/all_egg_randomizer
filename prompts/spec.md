# 用户需求

- 可以用鸡蛋砸出各种物品，什么都可以砸出来，包括生物和物品。
- 需要添加一个命令来控制可以砸出什么类型的东西，比如控制砸出生物、物品、效果（药水效果等等）、事件（闪电等等），具有可扩展性。默认只有生物和物品启用。
- 砸出物品的随机算法是伪随机算法，这样可以让各种可能均等出现。需要设置每个分类的权重，可以进行调整，默认全部相等。
- 提供 UI 面板来调整砸出的分类，以及设置的权重。面板的操作 UI 按照功能实现进行合理编排，从用户使用需求角度进行 UI 编排，为提供更好的用户使用体验。

- 代码应易于维护、模块化且具有易读性，模块、文件等应遵守功能的单一职责原则，避免在同一个文件当中堆砌功能无关的代码。

- 在适当的实现时使用设计模式来进行合理的设计，以提高代码的可扩展性，但同时避免过度设计。
