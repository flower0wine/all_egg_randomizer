# 实体机制参考文档：彩虹羊 (Rainbow Sheep)

**适用环境**：Minecraft Java Edition 1.21.11 (包含 Fabric 模组加载器环境)
**实体类型**：普通绵羊 (`minecraft:sheep`)
**核心触发条件**：实体自定义命名 (`CustomName`) 设为 `jeb_`

## 1. 概述 (Overview)
在 Fabric 1.21.11 环境下，“彩虹羊”并非独立的生物种类，也不存在单独的实体 ID。它本质上是原版 Minecraft 自 1.7 版本保留至今的一个经典**命名彩蛋（Easter Egg）**。Fabric 核心 API 未对该原版机制进行任何底层修改，因此在不安装破坏性模组的前提下该机制 100% 生效。

**⚠️ 变种区分说明**：
*   **Java版彩虹羊 (当前)**：普通绵羊附带 `jeb_` 标签，16种颜色实时循环渲染。
*   **《Minecraft Earth》彩虹羊**：已停服，为独立的实体变种，具有固定的6种颜色材质。Java 版**不会**加载该 Earth 独占内容。

## 2. 技术实现与生成方式 (Implementation Methods)

### 2.1 生存模式 / UI 交互层面实现
玩家在标准生存流程中触发此状态的路径如下：
1. **所需物品**：命名牌 (`minecraft:name_tag`)、铁砧 (`minecraft:anvil`)。
2. **数据修改**：在铁砧中将命名牌的名称数据更改为 `jeb_`（**严格要求**：小写字母 `j`，末尾带下划线 `_`，无多余空格）。
3. **实体交互**：玩家手持修改后的命名牌，对 `minecraft:sheep` 实体触发 `Interact`（右键）事件。

### 2.2 控制台命令 / NBT 数据层面实现
开发者或管理员可通过修改实体的 NBT 数据直接生成或转化彩虹羊。

*   **直接召唤 (Summon)**：
    ```mcfunction
    /summon minecraft:sheep ~ ~ ~ {CustomName:'{"text":"jeb_"}'}
    ```
*   **修改现有实体 (Data Merge)**：
    获取最近的绵羊实体并覆写其 `CustomName` 标签：
    ```mcfunction
    /data merge entity @e[type=sheep,limit=1,sort=nearest] {CustomName:'{"text":"jeb_"}'}
    ```

## 3. 实体行为与特性 (Behaviors & Properties)

*   **材质渲染**：彩虹效果是基于客户端的**实时渲染**，而非修改羊毛本身的 NBT 数据。材质会在原版 16 种羊毛颜色之间平滑循环。
*   **状态更新**：
    *   **剪毛事件**：对彩虹羊使用剪刀，掉落的羊毛物品为**普通颜色羊毛**（颜色通常取决于羊被命名之前的原始底色或随机染色），**不会**掉落“彩虹羊毛”方块。
    *   **进食/再生**：羊吃草后羊毛再生，系统会继续读取 `jeb_` 命名并维持彩虹渲染效果。
*   **相关命名彩蛋**：将实体命名为 `Dinnerbone` 或 `Grumm` 会触发实体的倒立渲染模型（注：在原版机制中，实体通常只能应用一个命名标签）。

## 4. 模组兼容性与开发建议 (Mod Compatibility & Development Notes)

1. **核心兼容性**：Fabric API 默认完全兼容此特性。
2. **潜在冲突排查**：如果生成后彩虹效果不生效，需检查以下模组类型：
    *   **覆盖绵羊渲染逻辑的模组**（如 `SheepVariety` 或其他动物模型修改模组）。
    *   **修改命名牌机制或实体 NBT 加载逻辑的模组**。
3. **扩展开发/资源包利用**：
    *   **资源包 (Resource Packs)**：可以通过安装特定资源包（如 `Rainbow Jeb Wool`），将 `jeb_` 命名逻辑扩展到**羊毛方块**上，实现放置方块的变色渲染。
    *   **模组增强**：开发者可编写 Fabric 扩展模组（如 `jeb_ Wool`），将客户端渲染的彩虹效果实体化，使其能够掉落真正的“彩虹羊毛”物品。
