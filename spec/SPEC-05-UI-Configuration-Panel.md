# SPEC-05: UI 配置面板系统

## 1. 需求概述

实现游戏内配置面板（GUI），允许玩家通过图形界面调整鸡蛋随机产物的分类启用/禁用状态和各分类的权重，提供良好的用户体验。

## 2. 预期效果

### 2.1 功能预期
- 通过命令 `/allegg gui` 打开配置面板
- 面板显示所有产物分类及其当前配置
- 可在面板中切换分类的启用/禁用状态
- 可在面板中调整各分类的权重
- 配置修改后实时生效

### 2.2 用户体验
- UI 布局清晰，按功能模块分组
- 权重调整使用滑动条或数值输入
- 启用/禁用使用开关控件
- 提供"保存"和"取消"按钮
- 面板关闭后自动保存配置

## 3. UI 设计

### 3.1 面板布局

```
+------------------------------------------+
|         All Egg Randomizer 设置          |
+------------------------------------------+
|  [分类列表区]          |  [编辑详情区]    |
|  +--------------+     |                  |
|  | 生物 (ENTITY)|     |  当前选择: 生物   |
|  | 物品 (ITEM)  |     |  ---------------  |
|  | 效果 (EFFECT)|     |  [启用开关]       |
|  | 事件 (EVENT) |     |  权重: [====--]  |
|  +--------------+     |  权重值: 1.0    |
|                       |                  |
|                       |  [保存] [重置]   |
+------------------------------------------+
|  [全局设置]                                |
|  最大生成数: [=====] 5                    |
+------------------------------------------+
```

### 3.2 UI 组件说明

#### 分类列表组件
- 显示所有产物分类
- 左侧显示分类图标/名称
- 右侧显示启用状态指示（✓/✗）
- 选中项高亮显示

#### 权重编辑组件
- 滑动条: 范围 0.0 - 10.0，步进 0.1
- 数值输入框: 支持直接输入数值
- 实时显示当前权重值

#### 启用/禁用开关
- 切换按钮: 绿色表示启用，灰色表示禁用
- 点击即可切换状态

#### 全局设置区域
- 最大生成数量滑动条
- 刷新率/冷却设置（可选）

### 3.3 交互流程

```
1. 玩家执行 /allegg gui
2. 打开配置面板，显示当前配置
3. 玩家点击分类项 → 右侧显示编辑详情
4. 玩家调整权重 → 实时预览
5. 玩家点击启用/禁用 → 切换状态
6. 玩家点击保存 → 配置生效并保存到文件
7. 玩家关闭面板 → 自动保存
```

## 4. 系统设计

### 4.1 架构设计

#### 模块职责
- **EggConfigScreen**: 主配置界面类，继承 Minecraft GUI 类
- **CategoryListWidget**: 分类列表组件
- **WeightSlider**: 权重滑动条组件
- **ToggleButton**: 启用/禁用开关组件
- **ConfigGuiManager**: GUI 管理器，负责打开/关闭面板

#### 设计模式
- **MVC 模式**: UI 组件（View）与配置管理器（Model）分离
- **观察者模式**: 配置变更时通知相关组件更新

### 4.2 界面层级

```
Screen
└── EggConfigScreen (主界面)
    ├── ContainerWidget (容器)
    │   ├── CategoryListWidget (分类列表)
    │   └── EditPanelWidget (编辑面板)
    │       ├── ToggleButton (启用开关)
    │       ├── WeightSlider (权重滑块)
    │       └── NumberField (数值输入)
    ├── ButtonWidget (保存按钮)
    └── ButtonWidget (取消按钮)
```

### 4.3 数据流

```
用户操作 → UI组件 → ConfigManager → 保存配置文件
                    ↓
              更新内存配置
                    ↓
              通知 EggThrowEventHandler (热更新)
```

## 5. 技术要点

### 5.1 Fabric GUI 框架
- 使用 Fabric API 的 Screen 扩展
- 或使用 Minecraft 原生 GUI 组件

### 5.2 依赖模块
- 配置系统 (SPEC-02): 读取和保存配置
- 命令系统 (SPEC-03): 打开 GUI 的命令入口

### 5.3 界面样式
- 遵循 Minecraft 1.21.11 的 UI 风格
- 使用原版按钮、滑动条样式
- 确保在各种分辨率下正常显示

### 5.4 键盘/鼠标交互
- 支持鼠标点击和拖拽
- 支持键盘导航（Tab 切换焦点）
- ESC 键关闭面板

## 6. 验收标准

- [ ] GUI 面板能够正确打开和关闭
- [ ] 所有分类显示正确
- [ ] 启用/禁用切换功能正常
- [ ] 权重调整功能正常
- [ ] 配置保存和加载正确
- [ ] UI 布局符合设计
- [ ] 热更新生效（不重启游戏）
